# 故障排除

## 无法获得当前登录用户所在的群组或结果不完整

首先，在登陆问题用户账号的情况下，访问discourse的`/session/current.json`接口，找到`current_user.groups`数组。
这是后端判断当前用户是否属于某个组的依据，若此数组为空或缺少特定用户所在的群组，会影响判断，请按以下方法修复：

1. 登录discourse管理员后台，进入群组设置，找到问题群组。
2. 在"管理"->"互动"里，将"谁能看到此群组"至少设置为"群组所有者、成员和版主"。
3. 再次访问`/session/current.json`接口，查看`current_user.groups`数组，问题应该已经解决。
4. 对每个oauth系统要用到的群组都要进行同样的设置。
